# 服务器上保持后台训练（关本地电脑后继续）

## 先弄清一件事

训练是在**远端服务器**上跑的。你关掉**自己家的电脑**或断开 **SSH**，只要**服务器实例还在**、**训练进程没有被一起结束**，训练就会继续。

若你是用 SSH 登录后直接在终端里执行 `python train_optimized.py`，**一关 SSH 窗口，进程往往会被系统挂断（SIGHUP）而退出**。所以要下面几种方式之一，让进程和终端「脱钩」。

---

## 方式一：tmux（推荐，可随时回去看进度）

```bash
# 1. 进入项目目录
cd ~/RT-detr

# 2. 新建一个会话（名字随意）
tmux new -s train

# 3. 在 tmux 里正常启动训练
python train_optimized.py

# 4. 断开会话但保持训练：按 Ctrl+B，松开后按 D（detach）
#    之后可以关本地电脑、关 SSH 客户端
```

之后想再看终端输出：

```bash
tmux attach -t train
```

结束训练：在 tmux 里按 `Ctrl+C`，然后输入 `exit` 退出会话。

列出所有会话：`tmux ls`

---

## 方式二：screen

```bash
screen -S train
cd ~/RT-detr && python train_optimized.py
# 断开会话：Ctrl+A，再按 D
```

重新连接：`screen -r train`

---

## 方式三：nohup（简单，日志进文件）

```bash
cd ~/RT-detr
nohup python train_optimized.py > train.log 2>&1 &
echo $!   # 记下进程号，便于排查
```

看实时日志：

```bash
tail -f train.log
```

---

## AutoDL / 云平台注意

- 不要点「关机」「释放实例」，否则服务器没了，训练也会停。
- 计费与实例是否运行有关，与你是否关家里电脑无关。
- 若平台提供「无卡模式」或长期关机策略，以平台说明为准。

---

## 小结

| 做法           | 关 SSH / 关本地电脑后 |
|----------------|------------------------|
| 直接在前台跑   | 通常会中断             |
| tmux / screen  | 一般会继续             |
| nohup … &      | 一般会继续             |

优先用 **tmux**：既能后台跑，又能随时 `attach` 看输出、打断点。
